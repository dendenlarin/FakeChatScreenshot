# Contributing

Спасибо за интерес к проекту! Чтобы участие было простым и предсказуемым:

- **Запуск**: `npm install`, затем `npm run dev`. Перед PR убедитесь, что проходят `npm run lint` и `npm run build`.
- **Стиль**: используем TypeScript, Tailwind и shadcn/ui. Старайтесь переиспользовать компоненты из `components/ui` и избегать инлайн-стилей там, где есть готовые токены/классы.
- **Тесты/проверки**: если добавляете новую утилиту или меняете экспорт, покрывайте юнит-тестом или добавляйте маленький e2e в описании PR (скрин/видео в привет).
- **Аксессибилити**: иконки без текста — с `aria-label`, сохраняем видимый фокус.
- **Коммиты/PR**: маленькие и тематические. В описании — что поменяли, зачем и как проверить.
- **Дизайн**: при добавлении нового мессенджера придерживайтесь каноничных отступов/цветов и общей высоты телефона (сейчас 620px).

## Как предложить изменения

1. Форк / ветка из `main`.
2. Внесите правки, добавьте тесты (если нужно).
3. Запустите проверки: `npm run lint` и `npm run build`.
4. Откройте PR с кратким описанием проблемы и решением. Скриншот превью — бонус.

## Сообщить об ошибке

Откройте issue с кратким воспроизведением: браузер/ОС, выбранный мессенджер, конфиг сообщений, что ожидали и что получили. Скриншоты приветствуются.
